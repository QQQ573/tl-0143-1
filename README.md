# 拼豆立体作品预览工具 · Perler Bead 3D Previewer

基于 Three.js r160 + TypeScript + Vite 的浏览器端实时预览工具。帮助学员在设计阶段发现重心前倾风险，避免重色块堆叠顶层导致晾干后作品变形。

## ✨ 功能特性

- 🧱 **立方体拼豆单元** — 基础 BoxGeometry mesh，按色号着色
- 📚 **分层编辑** — 每层 N×N 网格（默认 4×4，可配置 3×3 ~ 8×8）
- 🎨 **色号面板** — 内置 12 色常用色号（H01~H12），含权重系数
- ⚖️ **实时重心检测** — 计算整体重心投影点并与底座范围对比
- 🟠 **风险警示** — 重心超出底座 15% 时，风险拼豆单元自动标橙
- 🖱️ **交互** — 拖拽旋转 / 滚轮缩放 / 移动端双指捏合
- 🔩 **底座切换** — 钥匙扣 / 摆件 / 手机支架 三种底座 mesh
- 👁️ **层显隐** — 隐藏某层查看内部串珠通道
- 🔗 **挂点标记** — Shift+点击标记钥匙扣挂点位置
- 📦 **Docker 一键部署** — nginx + gzip 静态托管

## ⚖️ 重心计算公式

### 符号定义

| 符号 | 含义 | 单位 |
|------|------|------|
| $n$ | 拼豆总数 | — |
| $w_i$ | 第 $i$ 颗拼豆的色号权重系数（见色号权重表） | g |
| $(x_i, y_i, z_i)$ | 第 $i$ 颗拼豆的世界坐标（以作品几何中心为原点） | mm |
| $(X, Y, Z)$ | 整体重心坐标 | mm |

### 加权平均公式

$$
X = \frac{\sum_{i=1}^{n} w_i \cdot x_i}{\sum_{i=1}^{n} w_i}, \quad
Y = \frac{\sum_{i=1}^{n} w_i \cdot y_i}{\sum_{i=1}^{n} w_i}, \quad
Z = \frac{\sum_{i=1}^{n} w_i \cdot z_i}{\sum_{i=1}^{n} w_i}
$$

### 坐标计算

设网格尺寸为 $G$，层号 $l$，行号 $r$，列号 $c$（均从 0 开始），拼豆步长 $s = 1.05$，层高 $h = 1.05$：

$$
x_i = (c - \frac{G-1}{2}) \cdot s, \quad
y_i = l \cdot h, \quad
z_i = (r - \frac{G-1}{2}) \cdot s
$$

### 稳定判定

设底座半宽 $B_w$、半深 $B_d$，阈值系数 $k = 1.15$：

$$
|X| \le k \cdot B_w \quad \text{且} \quad |Z| \le k \cdot B_d
$$

超出则橙色标记 80% 外缘区域内的拼豆单元。

## 🔗 挂点命名规范

钥匙扣挂点使用 `HANGER-{LAYER}-{ROW}-{COL}` 格式命名。

### 命名规则

```
HANGER-{L}-{R}-{C}
   │     │    │    │
   │     │    │    └── 列号 (0-based, 十进制, 无补零)
   │     │    └─────── 行号 (0-based, 十进制, 无补零)
   │     └──────────── 层号 (0-based, 十进制, 无补零)
   └────────────────── 固定前缀
```

### 示例

| 名称 | 含义 |
|------|------|
| `HANGER-0-1-2` | 第 1 层（底层），第 2 行，第 3 列 |
| `HANGER-2-0-0` | 第 3 层，第 1 行，第 1 列（顶角挂点） |

### 最佳实践

1. **位置** — 挂点应放在顶层边缘或最上层两侧对称位置，避免单个挂点承重
2. **数量** — 建议 2~4 个挂点均匀分布，使挂绳合力线穿过重心正上方
3. **可见性** — 挂点必须贯穿至少 2 层拼豆的串珠通道，防止拉扯断裂
4. **坐标导出** — 导出 STL/打印时，挂点坐标 = 命名解析出的网格位置

## 🎯 性能指标

- 场景 polygon 上限：**≤ 36,000**（每颗拼豆约 12 三角面，8×8×10 层 = 640 颗 ≈ 7,680 tris，预留余量用于底座和辅助几何）
- 目标帧率：**1080p @ 60fps**（桌面端 Chrome）
- 渲染器：WebGL2，PCF 软阴影，`pixelRatio` 限制 ≤ 2
